Đặc tả Thanh toán (QR Code & VNPay)

1. Bảng dữ liệu

1.1 Bảng payments

payments - id (PK) - order_id - amount - currency (VND) - method (QR | VNPAY | CASH) - status (PENDING | SUCCESS | FAILED) - transaction_code - payment_url - qr_code - paid_at - created_at

1.2 Bảng payment_logs (khuyến nghị)

payment_logs - id - payment_id - type (WEBHOOK) - raw_data (JSON) - created_at

1.3 (Optional) Bảng payment_items

payment_items - id - payment_id - name - amount

2. API Thanh toán

2.1 Tạo thanh toán

POST /api/v1/payments { "order_id": 123, "amount": 500000, "method": "VNPAY" }

Response

{ "status": "success", "data": { "payment_id": 999, "payment_url": "https://sandbox.vnpay.vn/...", "qr_code": null } }

2.2 Thanh toán QR

{ "order_id": 123, "amount": 500000, "method": "QR" }

Response

{ "status": "success", "data": { "payment_id": 1000, "qr_code": "000201010212...", "bank_info": { "account_name": "CONG TY ABC", "account_number": "123456789", "bank": "VCB" } } }

2.3 Webhook từ cổng thanh toán

POST /api/v1/payments/webhook { "transaction_code": "VNP123456", "amount": 500000, "status": "SUCCESS", "order_id": 123 }

Xử lý backend

- Verify chữ ký (signature) - Update payments.status = SUCCESS - Lưu log vào payment_logs

2.4 Kiểm tra trạng thái thanh toán

GET /api/v1/payments/{payment_id} { "status": "success", "data": { "status": "SUCCESS" } }

3. Flow hoạt động

3.1 VNPay

1. App gọi POST /payments 2. Nhận payment_url 3. Mở WebView 4. User thanh toán 5. VNPay gọi webhook về server 6. Server update trạng thái 7. App gọi GET /payments/{id} để kiểm tra

3.2 QR Code

1. App gọi POST /payments 2. Hiển thị QR 3. User chuyển khoản 4. Backend nhận webhook hoặc xác nhận thủ công 5. App polling GET /payments/{id}

4. Lưu ý quan trọng

  • POST /payments chỉ tạo giao dịch, không phải thanh toán thành công
  • Luôn dùng GET /payments/{id} để kiểm tra trạng thái
  • Thanh toán là async (không đồng bộ)
  • Không trust client, phải verify từ backend
  • Nên lưu log webhook để debug & đối soát
